# ほぼ静的なページの作成

## テストを通して開発を進める

### テストの目的

- 機能停止になるような回帰バグの防止
- 安全なリファクタリングを行うため
- アプリケーションやインターフェースの設計に役立てる

### テストを書くタイミング

慣れてくればテストから書き始められる
仕様が明確に見えているのであれば｢先に｣書く
仕様が固まっていなければ｢後から｣書く
バグを見つけたときはまずバグを再現するためのテストを書く
コードがコロコロ変わりそうなものは｢後で｣テストを書く
リファクタリングする前にテストを書く

テストから書く場面)
    構造が単純
    仕様が明確に見えている
    バグを見つけた時
    リファクタリングする前

後からテストを書く場面)
    仕様がまだ固まっていない時
    設計がすぐに変更されそうな時
